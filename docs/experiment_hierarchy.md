## Иерархия следующих приоритетов (P0–P4)

```
P0: GPU layout
  — Morton и block-sparse layouts: неэффективны из-за overhead
  — Compact layouts: перспективны при low sparsity + large grid
  — Статус: частично исследован, результат негативный для Morton

P1: Tree compression
  — Сжатие ветвей дерева (segment compression)
  — Зависит от: стабильной структуры дерева

P2: Auto-tuning
  — Автонастройка порогов гейта (instability/FSR thresholds)
  — Сейчас пороги ручные

P3: Tree semantics
  — Кусты, LCA-расстояние как feature
  — Даёт ли дерево смысловую метрику?

P4: Downstream compatibility
  — «Не сломать фичи»: гарантия совместимости с потребителями
  — Неоднородная глубина уточнения vs. согласованность вектора признаков
```

## Более дальний горизонт (без конкретных дат)

Эти направления помечены как **преждевременные** без конкретных multi-objective use cases:

- **DAG-based routing** — маршрутизация уточнения по направленному графу вместо дерева
- **Wasserstein distance routing** — выбор маршрутов по расстоянию Вассерштейна
- **Segment compression** — сжатие ветвей дерева
- **Discrete signature systems** — дискретные сигнатуры для dirty detection

---

## Принцип планирования

Эксперименты структурированы с **явными kill criteria** перед запуском. Если эксперимент не проходит kill criteria — он закрывается, а не «допиливается».

Порядок выполнения: строго по иерархии P0->P1->P2->P3->P4. Нельзя прыгать вперёд без закрытия зависимостей.
