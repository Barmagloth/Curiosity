# Curiosity — Target Problem Definition (v1.1)

Документ фиксирует зачем проект существует, что считается успехом и против чего сравниваться.
Без этого "quality-per-cost" — туман в цилиндре.

Связан с: `curiosity_concept_v1.6.md` (как построено), `experiment_hierarchy.md` (что проверяется).

---

## Структура целей

У проекта две последовательных цели, а не параллельных. Порядок не опциональный.

```
Track A — построить инструмент
    ↓  (только после Instrument Readiness Gate)
Track B — исследовать инструментом
```

Track A может быть успешным, даже если Track B провалится.
Track B нельзя честно начинать, пока Track A не достиг инструментальной зрелости.
Иначе исследуются артефакты недоделанного прибора, а не структура мира.

---

## Track A — инструмент

### Вопрос
Можно ли построить систему, которая локально уточняет представление под бюджетом, не ломает границы, не слепнет без exploration и не разваливает downstream-полезность?

### Формулировка
Для задач с локально неоднородной информационной плотностью построить adaptive refinement pipeline, который:
- считает только информативные области,
- сохраняет качество не хуже dense baseline при меньшей стоимости,
- не создаёт артефактов на границах (halo),
- не становится структурно слепым (probe),
- удерживает расход под контролем (governor),
- не разрушает валидность представления при неоднородной глубине уточнения (scale-consistency).

### Domain
Задачи с пространственно неоднородной структурой:
- 2D/3D поля с резкими границами и плоскими областями
- Изображения с мультимасштабной структурой
- Volumetric / sparse scenes
- Multiscale signals с локальными аномалиями

### Success criteria
| Метрика | Требование |
|---|---|
| PSNR / MSE | ≥ dense baseline при equal budget |
| HF metric (interior-only) | Не хуже dense (нет seam-артефактов) |
| Cost compliance | StdCost / P95 в пределах governor target |
| Feature consistency (D_parent) | < τ_parent на ≥ 90% узлов |
| Winrate vs random selection | ≥ 90% по основной метрике |
| Overhead | Цена управления не съедает выигрыш |

### Cost model
Primary: число refined тайлов × стоимость refine операции.
Budget задаётся как target_per_step (тайлов/шаг). Overhead системы включён в бюджет (cost-fair).

### Baselines
1. **Dense baseline** — полный пересчёт на максимальном разрешении (оракул качества)
2. **Same-budget random** — тот же бюджет, случайный выбор тайлов
3. **Same-budget uniform coarse** — тот же бюджет, равномерное грубое приближение

### Статус
**Активная цель.** Exp0.1–Exp0.8 закрыты, базовые инварианты подтверждены.
Следующий шаг — Exp0.9b0 и baseline для scale-consistency.

## Instrument Readiness Gate

Track B начинается только после прохождения всех пяти критериев.

| Критерий | Что проверяется |
|---|---|
| **Invariant pass** | Все обязательные инварианты выполняются: halo, probe, governor, scale-consistency. Ни один не нарушается систематически. |
| **Overhead profile** | Измерен реальный overhead системы управления. Он не съедает выигрыш от adaptive refinement. Cost-fair сравнение показывает положительный результат. |
| **Stability pass** | Система устойчива: governor не осциллирует, split-решения воспроизводимы, результаты стабильны между прогонами. |
| **One validated benchmark** | Хотя бы один benchmark, на котором adaptive > same-budget random > coarse, с подтверждёнными числами и зафиксированным протоколом. |
| **Attribution diagnostics** | Понятно, какой модуль что даёт. Ablation или attribution показывает вклад halo, probe, governor, ρ по отдельности. Нет "чёрного ящика". |

**Без любого из пяти — Track B преждевременна.**

**С любыми пятью — Track B можно начинать, даже если:**
- governor ещё не лучший из возможных,
- ρ ещё не окончательно канонизирована,
- layout ещё не оптимален,
- скорость ещё далека от боевой.

Инструментальная зрелость ≠ совершенство. Это минимум, при котором прибор не врёт.

## Track B — исследование

### Вопрос
Несёт ли структура дерева уточнений собственный смысл — или это просто бухгалтерия сплитов?

### Формулировка
Использовать построенный механизм как прибор для изучения того, даёт ли дерево уточнения содержательную геометрию представления.

### Research questions
1. Коррелирует ли LCA-расстояние в дереве с семантическим расстоянием в пространстве задачи?
2. Несут ли кусты (множества путей к одному смыслу) реальную различимость — или это артефакт ρ?
3. Является ли карта смысловой плотности (distribution of splits) предсказательной для downstream-задач?
4. Является ли Scale-Consistency Invariant достаточным условием сохранения downstream-полезности — или нужны дополнительные ограничения?
5. Оживает ли phase schedule при неидеальном refine, где фазовая смена сигналов физически оправдана?

### Success criteria
- LCA-расстояние коррелирует с содержательной структурой данных (измеримо, не декларативно)
- Метрики дерева что-то реально объясняют или предсказывают в downstream-задаче
- "Смысловая геометрия" falsifiable: есть конкретные условия при которых гипотеза отвергается

### Domain
Определяется после достижения Instrument Readiness Gate, исходя из того какой downstream-потребитель наиболее естественен для построенного инструмента.

### Статус
**Исследовательская цель, не текущая.** Открытые вопросы зафиксированы в `curiosity_concept_v1.6.md` §13.

## Что сейчас честно не зафиксировано

| Вопрос | Статус |
|---|---|
| Конкретная downstream-задача для Track B | Не выбрана. Выбирается после Readiness Gate. |
| Feature consistency metric (downstream) | Не определена. D_parent — прокси, не прямая мера. |
| Применимость к нон-пространственным доменам (графы, латент) | Теоретически заявлена, экспериментально не проверена. |
| Общность за пределами пространственных данных | Долгосрочная амбиция, не текущая цель. |

## Одна фраза

**Curiosity — механизм адаптивного уточнения представлений под бюджетом, который решает где считать, считает только локально нужное, сохраняет корректность на границах и не допускает структурной слепоты или разрушения признаков.**

Сейчас строится инструмент (Track A). Потом им исследуется структура (Track B).
