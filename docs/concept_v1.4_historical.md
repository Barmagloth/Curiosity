# 0. Исходная мотивация

Цель проекта — не ускорить нейросеть как таковую и не заменить оптимизатор.

Цель — изменить пространство поиска так, чтобы:

* его можно было исследовать адаптивно,
* вычислительный ресурс тратился только там, где это даёт информацию,
* при этом сохранялись взаимосвязи признаков (не «ломались фичи»).

Ключевая идея:

> Размерность — это не фиксированное число, а глубина уточнения. Уточнение должно происходить только там, где это оправдано информационно и бюджетно.

# 1. Базовая модель

## 1.1 Пространство

Есть пространство состояний X (любой природы: признаки, латент, активации, данные).

Есть функция информативности ρ(x).

ρ(x) — мера того, где уточнение имеет смысл.

## 1.2 Уточнение

Процесс:

1. Стартуем с грубого приближения (coarse уровень).
2. Вычисляем ρ.
3. Решаем, стоит ли делить регион (split).
4. Уточняем только выбранные регионы.
5. Повторяем до остановки.

Уточнение всегда имеет форму:

```
output = coarse + delta
```

где delta:

* инициализируется нулём,
* ограничена по энергии,
* при отключении любого текущего уровня возвращает валидное представление предыдущего уровня,
* может ухудшать локальную согласованность у границ.

Halo применяется на каждом уровне уточнения.

# 2. Деревья и «кусты»

## 2.1 Дерево

Дерево — журнал маршрутов уточнения.

Каждый путь от корня к листу — последовательность решений split.

Структура должна быть GPU-дружественной (плоская упаковка, без pointer chasing).

## 2.2 Куст

Куст = множество путей, ведущих к одному смыслу.

Для первого прототипа используется метрика расстояния в дереве (через LCA / общий префикс).

---

# 3. Функция информативности ρ(x)

Семантика дерева определяется ρ.

ρ может отражать:

* ошибку задачи,
* variance,
* disagreement,
* компрессируемость,
* teacher-guided сигнал.

ρ может быть иерархическим.

---

# 4. Политика интересности

Баланс сигналов зависит от глубины L.

## 4.1 Абсолютная мощность

На coarse уровнях приоритет у крупных структур.

Вес абсолютной мощности затухает с глубиной.

## 4.2 Локальная аномальность

Чувствительность к локальной дисперсии усиливается с глубиной.

## 4.3 Окупаемость

Split допускается только если выигрыш перекрывает стоимость.

## 4.4 Бюджет

Жёсткие ограничения:

* max_depth
* max_nodes
* max_cost
* max_boundary_budget

## 4.5 Автонастройка строгости

Контроллер работает по EMA расхода.

## 4.6 Детектор мусора

Если средний выигрыш за K сплитов < δ (5–10% от предыдущего уровня) — дробление шума подавляется.

# 5. Boundary-aware refinement (обязательное требование)

Эксперимент Exp0.3 показал:

Refinement без overlap / feather / halo при тайловой подстановке гарантированно создаёт ложные high-frequency артефакты при разреженном бюджете.

Причина: Жёсткая вставка refined-тайла создаёт ступеньку на границе refined / non-refined. HF-метрики фиксируют это как искусственный сигнал.

Чем умнее adaptive выбор тайлов, тем сильнее артефакт.

Вывод: Halo / overlap — необходимое условие корректности.

Минимальное правило:

* overlap ≥ 3–4 ячейки (пикселя в image-экспериментах) при tile_size=16
* blending выполняется относительно coarse

Без boundary-aware refinement система наказывается high-frequency метриками при малых бюджетах.

---

# 6. Probe-budget exploration (обязательное требование)

Стратегия «уточнять только найденные границы» структурно слепа.

Возможна внутренняя структура домена, не проявляющаяся на текущем уровне.

Каждый домен хранит:

* фиксированный набор внутренних probe-плиток
* внутренний score
* счётчик давности проверки

На каждом шаге:

* 90–95% бюджета → exploitation
* 5–10% бюджета → probe

Приоритет probe:

* coarse residual / variance
* неопределённость
* давность проверки

Если probe выявляет активность — домен активируется.

Overlap не заменяет exploration. Exploration не лечит швы. Оба механизма обязательны.

---

# 7. Границы

Граница — зона затухания информативности.

Halo применяется по обе стороны границы.

---

# 8. Стратегии построения карты

1. Общая карта плотности.
2. Смысл-специфическая карта.

---

# 9. Ключевые инварианты

1. Уточнение аддитивно.
2. Halo обязателен.
3. Exploration обязателен.
4. Стоимость ограничена бюджетом.
5. Система не создаёт искусственные швы.
6. Система не становится структурно слепой.

---

# 10. Главный принцип Curiosity

Curiosity — система, которая:

1. Считает только там, где есть сигнал.
2. Делает это без создания искусственных швов.
3. Не становится слепой к внутренней структуре.
4. Управляет бюджетом осознанно.

Формально:

Refinement must be boundary-aware and must include controlled exploration.

# 11. Экспериментальный статус (Exp0.2–Exp0.3)

Подтверждено:

* Adaptive выбор тайлов даёт выигрыш по RGB MSE/PSNR.
* HF-деградация вызвана seam-артефактами.
* Halo ≥ 3–4 ячейки (пикселя в image-экспериментах) устраняет HF-деградацию.
* Interior-only HF подтверждает, что проблема была в швах.
